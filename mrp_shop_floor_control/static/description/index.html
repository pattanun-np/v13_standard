Shop Floor Control

New features implemented in version 13:

·       Recording in workorder confirmation phase of the following times:

o   Setup duration (actual start time)

o   Teardown duration (actual end time)

o   Working duration (actual working processing time)

o   Overall duration (as sum of previous ones to calculate the effective duration)

·       Managing partial confirmation quantities and manufacturing order closure

·       Entering actual by products quantities in workorder confirmation phase

·       Automatic creation of finished product’s lot when confirming workorders in progressive logic way

 

The following main features are provided for supporting the shop floor control process:

·       Workorder Scheduling and Work Center Capacity Loading

·       Capacity Leveling with mid-point Scheduling

·       Capacity Evaluation

·       Workorder Confirmation and Actual Dates Determination

·       Production Order Closure

 

Workorder Scheduling and Work Center Capacity Loading

·       when scheduling production order, workorders Start and End schedule dates are determined based on production order start date and times set in routing; forward scheduling logic has been implemented

·       parallel operations (operations with the same sequence in routing) are taken into account; hence parallel operations are scheduled in overlapping with the same start scheduled date

·       tool for rescheduling production order has been provided, so scheduled dates are calculated based in the new production order start date

·       work center capacity loading based on scheduled dates

 

Routing

·      

sequence for each activity has to be entered for sequencing properly production operations, i.e. workorders, in production order scheduling

 

Production Order

·      

Scheduled dates are determined in scheduling forward mode starting from start date; the cycle time entered in routing is taken into account as well as time before production and time after production; the working calendar set at WorkCentre level is used for calculating the schedule end date; it is possible to perform this activities several times as long as production order is in planned state.

 

·       scheduled start and end dates are determined on production order level based on scheduling at workorders level starting from production order start date, i.e. deadline start date, in forward mode

·       Rescheduling feature is provided for aligning scheduling dates to a new production order start date

 

Workorder Scheduled Dates


·       Scheduled dates determination based on times set in routing in forward scheduling logic starting from production order start date

·       the workorder scheduling is based on the following times:

◦     cycle time for each activity in routing

◦     time before production in the work center for each activity in routing

◦     time after production in the work center for each activity in routing

·       the scheduling is performed according the working calendar entered in the Work Center for each activity in routing

 

Capacity Loading


·       Workorder capacity requirement is calculated based on working hours determined in the scheduling phase; the capacity requirements are to be compared with the available capacity in the subsequent capacity leveling phase

·       Capacity requirements are

◦     determined at workorder creation and

◦     moved when changing workorder scheduled start date and

◦     closed when confirming workorder

·       All the concerned capacity requirements are deactivated when cancelling production order

 

Capacity Leveling with Mid-Point Scheduling


·      
Capacity leveling is achieved by moving workorders over the time so the corresponding capacity requirement is moved also

·       For moving workorder, the mid-point scheduling logic has been implemented for determining new scheduled dates for all the remaining active workorders, i.e. in ready, and pending status


·       Therefore, all the remaining active workorders are moved in backward or forward scheduling according to if they are preceding or following the concerned workorder

·       All the capacity requirements are moved also

·       Workorder in progress are not moved

·       Closed and cancelled workorders are not taken into account

·       Check whether the new Mid-Point scheduling is not compliant the start date of workorders in progress: the scheduling will be interrupted and the old scheduled dates will be restored

·       Parallel workorders, with the same sequence, are taken into account also, so scheduled with the same start date in forward scheduling logic or with the same end data in backward scheduling logic

 

Capacity Evaluation


·       Capacity leveling effectiveness can be checked by comparing the weekly available capacity with the capacity requirements over the time, week by week, at Work Center level

·       The work center capacity load and the remaining capacity are calculated for supporting the subsequent capacity leveling cycle

 

Workorder Confirmation and Actual Dates Determination

·       new tool is provided for recording times and quantities in workorder confirmation subprocess; consumption quantities are to be entered also

·       the following times can be entered in workorder confirmation:

◦     working duration

◦     setup duration (actual time before production in odoo terminology)

◦     teardown duration (actual time after production in odoo terminology)

therefore, the effective actual overall duration is calculated (the stardard odoo provides the elapsed one only)

·       workorder actual dates determination based on times declaration

·       production order actual dates determination based on workorder actual dates (on times declaration)

·       milestone activity can be confirmed and all the preceding workorders are closes, i.e. canceled

·       no milestone activity can be confirmed if any preceding workorder is in progress status

 

Workcenter


•       indicator to allow partial quantity confirmation (without setting it, the overall production quantity has to be declared in workorder confirmation)

•       indicator to allow starting workorder without component availability (without setting it, an error message is issued in phase confirmation)

 

Routing


·       an operation can be set as milestone

·       check no other operation can have the same sequence number, so no operation can be in overlapping to a milestone one

 

Production Order

•      

Actual dates are determined based on times declaration in Phase Confirmation process: the first start date for the first workorder and the last end date for the last workorder are determined

 

Workorder


·       actual dates determination based on confirmed times

·       all actual duration are tracked in time traction section; the overall duration is determined (the odoo standard provides the elapsed one only)

 

Workorder Confirmation


·       the following actual duration can be entered:

◦     working duration

◦     setup duration (actual time before production in odoo terminology)

◦     teardown duration (actual time after production in odoo terminology)

therefore, the effective actual overall duration is calculated (the stardard odoo provides the elapsed one only)

·       It is possible to enter the lot for finished product and for its components also.

·       Feature for automatic creation lot of finished product

·       Check to forbid partial quantity confirmation

·       check to avoid starting workorder without components availability

·       check to avoid to confirm produced quantity more than production quantity

·       check to produce quantity already produced by the previous workorder; for milestone activity this is not checked because it is possible to confirm a milestone activity and closing all preceding ones

·       actual components consumption quantities

·       actual by products confirmation quantities

 

Production Order Closure


·       New feature for closing manufacturing order with partial quantities and/or goods movements pending for partial consumption quantities.
